# V1.6 Harness Engine

The Harness Engine is the WorkContext-native execution path for software changes. It builds repository context, fingerprints the environment, validates structured edits, applies patches through an atomic applier, runs validation commands in a sandbox, records trajectory, produces artifacts, and gates completion on evidence.

Public entrypoints are `HarnessExecutionRequest`, `execute_harness_task`, `HarnessWorkContextService`, and the `/api/work-contexts/:id/harness/*` routes.
