# PrometheOS Lite - Memory Policy

## Remember
- Project goals and objectives
- User preferences and settings
- Generated architecture decisions
- Important context about the project
- Code patterns and conventions used
- Dependencies and their purposes

## Do Not Remember
- Secrets and API keys
- Private credentials (passwords, tokens)
- Temporary test data
- Personal information
- Sensitive configuration values
- Anything that could compromise security

## Memory Retention
- Long-term: Project context, architecture decisions
- Medium-term: Recent conversations, current tasks
- Short-term: Temporary context during active sessions

## Privacy First
All memory storage is local-first. No data is sent to external services without explicit user consent.
